Central Casting in practice

How the aimez.ai research program runs on the method.

This page shows the demo catalogue and tooling behind the aimez.ai research program, as a worked counterpart to the sanitized demo. The operating layer is Cursor.

The four columns

The program is organized along four columns that the catalogue keeps separate, so any file or action resolves to a clear place.

Instrumentation

The tool layers that produce work, from open-ended reasoning to in-editor drafting.

Actors

The work lanes, each a role with one job, indexed under the O lane.

Contracts

The system surfaces and written handoffs that say what each lane reads and writes.

ID

The stable index that names each lane by number, letter and roman numeral.

Instrumentation: the drafting layers

Open-ended planning runs in a general chat so heavy orchestration reasoning stays off the in-editor token budget. Cursor is the operating layer that reads and writes the repository. GitHub Copilot drafts inline inside the editor. Every layer is grounded in repository source, so the work stays anchored to what is committed.

Contracts: surfaces and handoffs

Every file carries a surface class so a reader knows from the path alone whether it is an authoritative memo, a derived report, a narrative log or current state. Each work lane keeps dated task homes named yyyymmdd_slug, and each home holds a manifest, a readme, a step log and written handoffs for the orchestrator, the research role and the runner. A session reads the step log first, so it resumes with the lane history intact.

Honest scope

The coordinating method was formalized in spring 2026, so it is recent and parts of it are still settling into habit. The discipline reduces context loss across long work and it leans on care more than on automation, so the honest claim is modest: it helps, and it is still being proven.
